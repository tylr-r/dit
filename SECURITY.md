# Security Policy

Dit is a solo-maintained project. This document explains how to report a
security issue and what to expect in return.

## Supported versions

Only the current `main` branch and the most recently shipped version of the iOS
app and [practicedit.com](https://practicedit.com) receive security fixes.
Older releases are not backported.

## Reporting a vulnerability

Please email **tyler@tylerobinson.com** with:

- A description of the issue and its impact
- Steps to reproduce (proof of concept welcome)
- The affected surface: this repo, the iOS app, the web app, or the Firebase
  backend

Do not open a public GitHub issue for suspected security problems.

## What to expect

- Acknowledgement within a few days.
- An honest assessment of severity and whether it's in scope.
- A fix, mitigation, or explanation of why no action will be taken.

As a solo maintainer I can't promise a formal SLA. I'll work issues as quickly
as my schedule allows, prioritizing by real-world impact.

## Scope

**In scope**

- This repository's source code
- The iOS app and [practicedit.com](https://practicedit.com)
- Data handling and Firebase security rules owned by this project

**Out of scope**

- Vulnerabilities in third-party services (Firebase, Expo, GitHub, etc.) —
  report those upstream
- Social engineering, physical attacks, or attacks requiring a compromised
  device
- Missing rate limits on non-sensitive endpoints
- Reports generated by automated scanners without a demonstrated impact

## Rewards

There is no bug bounty. Thanks and credit in release notes are available on
request for issues that lead to a fix.
